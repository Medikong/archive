
# 클라우드 네이티브 실습 통합 과제

## 0. 주요 역할군 분담안

이 분담안은 과제를 가능한 한 작은 책임 단위로 나눈 것이다. 각 역할은 담당 결과물을 끝까지 소유하며, 다른 역할의 결과물을 대신 결정하지 않는다. 팀 인원이 역할 수보다 적으면 아래의 `겸임 권장 묶음`을 기준으로 인접 역할을 합친다.

### 0.10. 겸임 권장 묶음

역할을 합쳐야 할 때는 아래 순서대로 인접 책임만 묶는다. 검증 역할은 가능하면 구현 역할과 다른 사람이 맡아 독립적으로 판정한다.

| 묶음 | 포함 역할 | 권장 담당 범위 |

| D. Kubernetes 플랫폼 | R15~R19 | 클러스터, 환경 분리, 워크로드, 패키징, 오토스케일링 |

| F. 메시·네트워크 | R26~R29 | 서비스 메시, 트래픽, mTLS, 복원력 정책 |

### 0.11. 공통 완료 기준

- 각 역할은 담당 산출물의 저장 위치, 재현 방법, 완료 판정 기준을 문서에 남긴다.
- 구현 결과만 제출하지 않고 실행 명령, 입력 조건, 버전, 결과 로그 또는 화면 캡처를 함께 보관한다.
- 설계 변경이 발생하면 영향을 받는 계약, 배포 선언, 대시보드, 테스트, 런북을 함께 갱신한다.
- 검증 결과에는 성공 사례뿐 아니라 실패 조건과 확인하지 못한 범위도 기록한다.
- R50은 모든 역할의 결과물을 새로 작성하지 않고, 역할별 완료 상태와 인계 누락만 관리한다.
