# DropMong 아키텍처 보드 디자인 시스템

## 1. 분위기와 정체성

DropMong 설계 보드는 어두운 운영 관제실처럼 보여야 한다. 핵심 흐름은 조용하고 선명하게 드러나고, 위험 지점은 과장 없이 즉시 눈에 들어와야 한다. 시그니처는 "드롭 경로 추적"이다. 사용자가 들어와 주문이 확정되기까지의 경로를 얇은 라인, 단계 카드, 상태 배지로 반복해서 보여준다.

## 2. 색상

### 팔레트

| 역할 | 토큰 | 값 | 사용처 |
| --- | --- | --- | --- |
| 기본 표면 | `--surface-primary` | `#08090a` | HTML 본문 배경 |
| 패널 표면 | `--surface-panel` | `#101114` | 주요 패널 |
| 돌출 표면 | `--surface-raised` | `#17191f` | 카드와 다이어그램 블록 |
| 내부 표면 | `--surface-inset` | `#0c0d10` | 코드와 지표 영역 |
| 기본 텍스트 | `--text-primary` | `#f7f8f8` | 제목과 강조 라벨 |
| 보조 텍스트 | `--text-secondary` | `#c9d0dc` | 본문 |
| 흐린 텍스트 | `--text-muted` | `#8a91a0` | 캡션과 보조 라벨 |
| 약한 테두리 | `--border-subtle` | `rgba(255,255,255,0.06)` | 낮은 대비 구분선 |
| 강한 테두리 | `--border-strong` | `rgba(255,255,255,0.13)` | 중요한 패널 |
| 기본 강조색 | `--accent-primary` | `#6f7cff` | 링크, 포커스, 핵심 경로 |
| 부드러운 강조색 | `--accent-soft` | `rgba(111,124,255,0.16)` | 강조 배경 |
| 성공 | `--status-success` | `#29c46a` | 검증 완료와 정상 상태 |
| 경고 | `--status-warning` | `#f0b35a` | 설계 또는 전환 필요 상태 |
| 오류 | `--status-error` | `#ff6b6b` | 치명적 위험 |
| Kafka | `--kafka` | `#c087ff` | 이벤트 버스와 비동기 흐름 |
| 인프라 | `--infra` | `#5bd6d0` | 플랫폼과 ingress 제어 |
| 데이터 | `--data` | `#ffd166` | 데이터베이스, outbox, cache |

### 규칙

- 핵심 checkout 경로에는 대표 강조색 `--accent-primary` 하나를 사용한다.
- `--kafka`, `--infra`, `--data`는 의미가 있는 다이어그램 색상으로만 사용한다.
- 장식용 색상 블록은 사용하지 않는다. 모든 색상은 서비스, 데이터, 인프라, 위험, 검증 상태 중 하나를 의미해야 한다.

## 3. 타이포그래피

### 크기 체계

| 단계 | 크기 | 굵기 | 줄 높이 | 사용처 |
| --- | --- | --- | --- | --- |
| 디스플레이 | `clamp(40px, 7vw, 72px)` | 650 | 0.98 | HTML hero 제목 |
| H1 | `36px` | 620 | 1.1 | Markdown 페이지 제목 |
| H2 | `28px` | 610 | 1.2 | 주요 섹션 |
| H3 | `20px` | 610 | 1.35 | 카드와 다이어그램 제목 |
| 큰 본문 | `18px` | 400 | 1.65 | 소개 문단 |
| 본문 | `16px` | 400 | 1.6 | 기본 문장 |
| 작은 본문 | `14px` | 400 | 1.5 | 캡션 |
| 라벨 | `12px` | 620 | 1.35 | 배지와 표 라벨 |
| 고정폭 | `13px` | 500 | 1.45 | API 경로, topic, repo 경로 |

### 폰트 스택

- 기본: `Inter, ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`
- Mono: `"SFMono-Regular", Consolas, "Liberation Mono", Menlo, monospace`

### 규칙

- HTML 보드 라벨은 조밀해도 되지만, 본문은 14px 이상으로 읽을 수 있어야 한다.
- 고정폭 글꼴은 서비스명, topic, 경로, metric, command에만 사용한다.
- 좁은 패널에서 음수 자간을 쓰지 않는다.

## 4. 간격과 레이아웃

### 기본 단위

모든 간격은 4px 단위에서 파생한다.

| 토큰 | 값 | 사용처 |
| --- | --- | --- |
| `--space-1` | `4px` | 좁은 inline 간격 |
| `--space-2` | `8px` | 배지와 chip 간격 |
| `--space-3` | `12px` | 조밀한 카드 padding |
| `--space-4` | `16px` | 기본 카드 padding |
| `--space-6` | `24px` | 섹션 내부 간격 |
| `--space-8` | `32px` | 다이어그램 그룹 간격 |
| `--space-10` | `40px` | 페이지 섹션 간격 |
| `--space-12` | `48px` | 주요 분할 간격 |
| `--space-16` | `64px` | hero와 섹션 리듬 |

### 그리드

- 최대 콘텐츠 너비: `1240px`
- 주요 HTML 레이아웃: 데스크톱 우선 grid를 사용하고, 좁은 viewport 대응은 깨진 렌더링을 막는 수준으로만 둔다.
- Markdown 레이아웃: 제목, 표, Mermaid 다이어그램, 연결된 SVG asset을 기준으로 한다.

### 규칙

- 중첩 카드는 피한다. 주요 섹션에는 패널을 쓰고, 반복 항목에만 카드를 사용한다.
- 작은 화면에서는 텍스트를 12px 미만으로 줄이기보다 다이어그램을 가로 스크롤 가능하게 둔다.

## 5. 컴포넌트

### 섹션 패널

- 구조: `section.panel > header + content`
- 변형: default, important, source
- 간격: `--space-6`에서 `--space-8`
- 상태: default만 사용
- 접근성: 각 패널 안에 heading을 둔다.
- 모션: HTML에서만 약한 opacity와 translate 진입 효과를 사용한다.

### 서비스 노드

- 구조: 서비스명, 역할, 소유 데이터, 핵심 API
- 변형: 동기 서비스, 비동기 consumer, 플랫폼
- 간격: `--space-4`
- 상태: default, hover, focus
- 접근성: 서비스명은 이미지가 아니라 텍스트로 표현한다.
- 모션: hover 시 테두리와 배경만 변한다.

### 흐름 레일

- 구조: 순서가 있는 node를 선분으로 연결한다.
- 변형: 요청 경로, 이벤트 경로, 배포 경로
- 간격: `--space-4`
- 상태: default
- 접근성: Markdown의 순서 목록으로 같은 내용을 제공한다.
- 모션: 기본적으로 사용하지 않는다.

### 사용자 여정 단계

- 구조: 단계 번호, 단계 제목, 사용자 행동, 시스템 응답
- 변형: 고객, 운영자, 플랫폼
- 간격: `--space-4`
- 상태: default, hover, 링크된 경우 focus
- 접근성: 각 단계는 순서 있는 텍스트로 읽을 수 있어야 한다.
- 모션: hover 테두리만 사용한다.

### 결정 배지

- 구조: 짧은 라벨과 상태
- 변형: P0, P1, P2, risk, invariant
- 간격: `--space-2`
- 상태: default
- 접근성: 색상은 반드시 텍스트 라벨과 함께 사용한다.
- 모션: 사용하지 않는다.

### 지표 타일

- 구조: metric 이름, 목표값, 소유자
- 변형: SLO, test, alert
- 간격: `--space-4`
- 상태: default
- 접근성: 값은 텍스트로 표시한다.
- 모션: 사용하지 않는다.

## 6. 모션과 상호작용

| 유형 | 시간 | 이징 | 사용처 |
| --- | --- | --- | --- |
| 마이크로 | `120ms` | `ease-out` | Hover와 focus 상태 |
| 표준 | `220ms` | `ease-in-out` | 섹션 표시 |
| 강조 | `420ms` | `cubic-bezier(0.16, 1, 0.3, 1)` | Hero 시각 요소 진입 |

규칙:

- `opacity`와 `transform`만 애니메이션한다.
- `prefers-reduced-motion`을 존중한다.
- 장식 목적으로만 아키텍처 라인을 애니메이션하지 않는다.

## 7. 깊이와 표면

전략: 혼합하되 절제한다.

| 단계 | 처리 | 사용처 |
| --- | --- | --- |
| 페이지 | `--surface-primary` | 본문 |
| 패널 | `--surface-panel`, 약한 테두리 | 섹션 패널 |
| 돌출 | `--surface-raised`, 강한 테두리 | Node와 지표 타일 |
| 내부 | `--surface-inset`, 내부 테두리 | 코드, topic, 데이터 저장소 |

규칙:

- 깊이는 주로 색조 차이와 얇은 테두리로 만든다.
- 그림자는 hero 시각 요소와 sticky navigation에만 허용한다.
- 모서리 반경은 카드 8px, 패널 12px, pill은 full radius를 사용한다.
