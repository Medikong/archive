# Docs

`docs/`는 특정 구현 repo에 바로 반영하기 전, 학습/실험/운영 검증 계획을 문서 묶음으로 보관하는 공간이다.

## 문서 묶음

| 문서 | 용도 |
| --- | --- |
| [cloud-native-deployment-validation/](cloud-native-deployment-validation/) | Istio, LGTM, Chaos Mesh 기준의 클라우드 네이티브 배포 전략 실험 계획과 결과 기록 템플릿 |

## 작성 기준

- 실제 실험 전 문서는 계획, 가설, 측정 항목, 판정 기준을 먼저 남긴다.
- 실험 결과는 템플릿을 복사해 서비스별로 작성하고, 원본 템플릿은 수정하지 않는다.
- 숫자가 필요한 항목은 빈칸으로 두더라도 단위와 수집 위치를 함께 적는다.
- 외부 자료는 문서 하단의 참고 자료에 링크로 남긴다.
