# KREAM 랭킹 화면 (급상승 전체 랭킹 / 월간 전체 랭킹)

- 회사: 크림(KREAM)
- 원문: https://kream.co.kr/content/ranking_all-rising , https://kream.co.kr/content/ranking_all-monthly
- 접근일: 2026-07-09
- 주제: 실시간 인기 랭킹 신호 구성, 랭킹 기간 구분
- 확인 방식: WebSearch 결과 스니펫 + 제품 화면 설명 기준 (엔지니어링 블로그 아님)

## 원문 확인 내용

- KREAM은 "급상승 전체 랭킹"(최근 3일 기준)과 "월간 전체 랭킹"을 별도 화면으로 제공한다.
- 랭킹은 조회, 관심(찜), 거래 데이터를 함께 반영한다고 설명된다.
- 랭킹탭에 신상 코너가 별도로 있어, 신규 상품의 인기를 기존 상품과 구분해 보여준다.

## 적용 아이디어

- "조회 + 관심(찜) + 거래"라는 다중 신호 조합은 DropMong의 "찜 + 조회수"(오픈 전), "소진율"(오픈 후) 이원화 설계와 방향이 같다는 근거로 쓸 수 있다.
- "최근 3일 급상승" 같은 짧은 기간 윈도우 랭킹은, DropMong의 `elapsed_minutes` 기반 정규화가 같은 문제의식(최근 활동 우선)을 다루고 있음을 뒷받침한다.
- 신상/기존 구분은 DropMong의 `SCHEDULED`(오픈 전) / `OPEN`(오픈 후) 상태 분리와 유사한 발상이다.

## 확인 필요

- 이건 기술 블로그가 아니라 제품 화면 설명 수준이라, 실제 가중치·집계 주기·어뷰징 방지 로직은 확인할 수 없다.
