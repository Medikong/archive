# 크림 (KREAM)

## 개요

- 구분: 국내 한정판 리셀(재판매) 거래 플랫폼
- 주요 서비스: KREAM 앱/웹
- 조사 초점: DropMong과 가장 가까운 도메인(한정판 거래)의 실시간 랭킹 구성 신호

## 핵심 설계 포인트

| 주제 | 확인한 내용 | 출처 |
| --- | --- | --- |
| 랭킹 신호 구성 | "급상승 전체 랭킹"을 조회, 관심(찜), 거래 데이터를 함께 반영해 제공 | [급상승 전체 랭킹](https://kream.co.kr/content/ranking_all-rising), [월간 전체 랭킹](https://kream.co.kr/content/ranking_all-monthly) |
| 랭킹 기간 구분 | "최근 3일 급상승"과 "월간" 랭킹을 별도로 제공 | 위 출처 |
| 카테고리/신상 구분 | 랭킹탭에 신상 코너를 별도로 두어 신규 인기 상품을 구분 노출 | 위 출처 |
| 내부 알고리즘 상세 | 공개 자료에서 확인 불가 (제품 화면 기준 확인) | - |

## 참고할 점

- KREAM이 랭킹을 "조회 + 관심(찜) + 거래" 세 신호의 조합으로 구성한다는 점은, DropMong이 설계한 "찜 + 조회수"(오픈 전) → "소진율"(오픈 후) 이원화 구조와 방향이 같다. 다만 KREAM은 오픈 전/오픈 후 구분 없이 상시 거래 플랫폼이라 우리처럼 드롭 상태 전환에 따른 신호 전환은 없다.
- "최근 3일 급상승"처럼 **짧은 기간 윈도우**로 랭킹을 계산하는 방식은, 우리의 `elapsed_minutes` 기반 정규화와 유사한 문제의식(오래된 누적치보다 최근 활동을 우선)을 보여준다.
- 월간/급상승을 분리해 제공하는 것은, 우리가 검토했던 "오픈 전/오픈 후 랭킹 분리"와 유사하게 하나의 랭킹으로 모든 목적을 담지 않는다는 설계 원칙을 재확인해준다.

## DropMong에 그대로 가져오면 안 되는 점

- KREAM은 상시 재판매 시장(시세 차트 기반)이라 "한정 수량 최초 판매 오픈"이라는 DropMong의 핵심 순간이 없다. 신호 구성 아이디어만 참고하고, 오픈 전/후 상태 전환 로직은 DropMong 자체 설계를 따른다.
- 정확한 가중치, 집계 주기, 어뷰징 방지 로직은 공개되지 않아 추정할 수 없다.

## 확인 필요

- "조회/관심/거래"를 각각 어떤 비율로 합산하는지, 실시간 반영 주기가 몇 분/초 단위인지는 공개 자료로 확인하지 못했다.
