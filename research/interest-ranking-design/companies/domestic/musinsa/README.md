# 무신사

## 개요

- 구분: 국내 패션 커머스
- 주요 서비스: 무신사 스토어
- 조사 초점: 실시간 인기/베스트 랭킹 서비스의 신호 구성과 공정성 원칙

## 핵심 설계 포인트

| 주제 | 확인한 내용 | 출처 |
| --- | --- | --- |
| 실시간 신호 구성 | 조회, 구매 등 실시간 활동 지표를 랭킹에 반영한다고 밝힘 | [무신사 스토어, 국내 최초 실시간 베스트 랭킹 서비스 제공](https://www.musinsa.com/content/mz/6622) |
| 공정성/어뷰징 방지 | 어뷰징과 조작 없이 공정한 경쟁을 랭킹 서비스의 핵심 가치로 제시 | [무신사 스토어, 국내 최초 실시간 베스트 랭킹 서비스 제공](https://www.musinsa.com/content/mz/6622) |
| 구체 구현 방식 | 검색 스니펫 기준으로는 확인 불가 | - |

## 참고할 점

- 실시간 인기 지표를 사용자에게 그대로 노출하는 기능은 "몇 명이 지금 보는지/사는지" 같은 즉각적 신호가 사용자 신뢰와 구매 결정에 영향을 준다고 제시한다.
- 랭킹의 신뢰성(공정성)을 서비스의 핵심 가치로 명시한 점은, DropMong도 조회수 집계에 최소한의 방어 장치(로그인 요구, 중복 방지)를 두어야 한다는 근거로 쓸 수 있다.

## DropMong에 그대로 가져오면 안 되는 점

- 무신사는 대형 패션 플랫폼으로 트래픽 규모, 구체적인 어뷰징 탐지 로직, 인프라 구성이 공개되지 않았다. DropMong의 초기 규모에 맞는 훨씬 단순한 구현(로그인 필수 + 시간 윈도우 dedup)으로 시작해야 하며, 무신사 수준의 정교한 방어 체계를 목표로 삼지 않는다.
- "N명이 보는 중" 같은 UI 노출 여부는 이번 조사로 결정할 수 없다. `REQ.A.07` 열린 질문으로 남겨둔다.

## 확인 필요

- 실제 신호 집계/저장 방식(Redis, 별도 스트리밍 파이프라인 등)은 공개 자료에서 확인하지 못했다.
- 어뷰징 탐지의 구체적 기준(계정, 디바이스, 행동 패턴)도 확인하지 못했다.
