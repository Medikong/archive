# YouTube

## 개요

- 구분: 해외 동영상 플랫폼
- 주요 서비스: YouTube 조회수 집계
- 조사 초점: "조회"를 어떤 기준으로 정의하고, 어뷰징을 어떻게 걸러내는가 — 이번 조사에서 가장 구체적인 자료

## 핵심 설계 포인트

| 주제 | 확인한 내용 | 출처 |
| --- | --- | --- |
| 조회 판정 기준 | 30초 이상 시청(20초 미만 영상은 끝까지 시청) 시 조회로 인정. 연속 시청이 아니어도 합산 30초면 인정 | [How Does YouTube Count Views?](https://uppbeat.io/blog/youtube/how-does-youtube-count-views) |
| 단계적 검증 | 우선 조회를 집계한 뒤, 인위적 증폭 여부를 품질 검사하고, 시청자 행동 패턴을 검증한 뒤 최종 조회수를 조정 | [How Does YouTube Count Views? (2026 Guide)](https://www.subsub.io/blog/how-does-youtube-count-views) |

| 어뷰징 탐지 | 머신러닝으로 봇 활동 패턴을 탐지 — 특정 지역/기기에서의 비정상적 조회 급증, 동일 IP 반복 조회 등을 표시 | 위 출처 |
| 사용자 단위 제한 | 동일 사용자 기준 24시간 내 최대 5회까지만 조회수로 인정 | [How Does YouTube Count Views? (2026 Guide)](https://www.subsub.io/blog/how-does-youtube-count-views) |
| 제작자 자기 조회 제한 | 영상 업로더 본인의 조회는 최대 3회까지만 인정, 나머지는 집계되지 않음 | 위 출처 |

## 참고할 점

- **"우선 집계 → 사후 검증 → 조정"** 3단계 구조는, DropMong의 조회수 dedup을 "요청 시점에 무조건 막기"(현재 설계: `SET NX EX`로 사전 차단)보다 유연하게 접근할 수 있는 대안을 보여준다. 다만 DropMong은 초기 스코프상 사전 차단(Redis dedup) 방식이 구현이 훨씬 간단해 그대로 유지하는 게 맞다.
- **"동일 사용자 24시간 내 최대 N회"** 패턴은 DropMong이 검토했던 "5분 dedup"보다 더 긴 윈도우 옵션이 실제로 쓰인다는 근거가 된다. 다만 YouTube는 재생이라는 반복 가능한 행위, DropMong은 상품 상세 조회라는 성격 차이가 있어 그대로 옮기지는 않는다.
- **동일 IP 반복 조회를 탐지 신호로 쓴다**는 점은, 지난 대화에서 검토했다가 "로그인 장벽으로 대체 가능하니 생략"하기로 한 IP 레이트리밋을, 향후 매크로 문제가 실제로 발생하면 다시 꺼낼 수 있는 옵션으로 남겨둘 근거가 된다.

## DropMong에 그대로 가져오면 안 되는 점

- YouTube 규모(머신러닝 기반 패턴 탐지, 300회 기준 freeze 등)는 대형 플랫폼 전용 인프라다. DropMong 초기 스코프에서는 이런 정교한 사후 검증 체계를 목표로 삼지 않는다 (`REQ.A.07.NFR-003`에 이미 "완전한 봇 탐지가 아니다"라고 명시함).
- "재생 시간 30초"라는 기준은 동영상 특화 지표라 페이지 조회(우리 도메인)에는 그대로 적용할 개념이 없다. 그러나 "단순 클릭이 아니라 일정 시간 이상 머물러야 조회로 인정"하는 아이디어 자체는 참고할 수 있다 (예: 상세 페이지 진입 후 일정 시간 체류해야 조회로 카운트).

## 확인 필요

- 이 자료들은 크리에이터 대상 설명 글이라 YouTube 공식 엔지니어링 블로그 수준의 구현 상세(정확한 ML 모델, freeze 로직의 시스템 구조)는 확인할 수 없다.
