# How Does YouTube Count Views?

- 회사: YouTube
- 원문: https://uppbeat.io/blog/youtube/how-does-youtube-count-views , https://www.subsub.io/blog/how-does-youtube-count-views
- 접근일: 2026-07-09
- 주제: 조회 판정 기준, 단계적 검증(freeze), 어뷰징 탐지, 사용자/제작자 단위 제한
- 확인 방식: WebSearch 결과 스니펫 기준 (크리에이터 대상 3자 설명 글, YouTube 공식 기술 블로그 아님)

## 원문 확인 내용

- 조회로 인정되려면 재생 시작 후 30초 이상 시청해야 한다(20초 미만 영상은 끝까지). 연속 시청이 아니라 여러 구간을 합쳐 30초 이상이어도 인정된다.
- 조회수 집계는 "우선 표면적 조회 집계 → 인위적 증폭 여부 품질 검사 → 시청자 행동 패턴 검증 → 최종 조정"의 순서로 진행된다.
- 조회수가 300에 도달하면 일시적으로 고정(freeze)해, 실제 사람의 정상 재생인지 확인하는 절차를 거친다. 이후에도 시스템은 최초 300건을 포함해 유입 조회를 계속 추적한다.
- 머신러닝 기반 패턴 인식으로 봇 활동을 탐지한다. 특정 지역/기기에서의 비정상적 조회 급증, 동일 IP에서 반복되는 조회 등이 탐지 신호로 쓰인다.
- 동일 사용자는 24시간 내 최대 5회까지만 조회수로 인정된다.
- 영상 업로더 본인의 조회는 채널을 통해 시청해도 최대 3회까지만 인정된다.

## 적용 아이디어

- DropMong의 조회수 dedup 설계(`SET NX EX 300 "viewed:{dropId}:{userId}"`, 5분 윈도우)는 YouTube의 "사전 차단"보다 단순한 "요청 시 즉시 차단" 방식이다. YouTube처럼 "일단 집계하고 사후에 이상 여부를 걸러내는" 방식은 더 정교하지만 구현 비용이 크므로, 1차 스코프에서는 현재의 사전 차단 방식을 유지하는 근거로 쓴다.
- "동일 IP 반복 조회 탐지"는, 로그인 장벽만으로 충분하다고 판단해 제외했던 IP 레이트리밋을 향후 재검토할 근거 자료로 analysis에 남겨둔다.

## 확인 필요

- 이 설명들은 크리에이터 대상 3자 해설 글로, YouTube의 실제 시스템 구조(어떤 DB/스트리밍 인프라를 쓰는지)는 확인할 수 없다.
